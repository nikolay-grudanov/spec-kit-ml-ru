# Спецификация данных: Cats vs Dogs Dataset

**Created**: 2026-02-13
**Status**: Draft
**Input**: Описание задачи: "Создать модель для классификации изображений кошек и собак"

## Описание датасета

### Общая информация
- **Название датасета**: Cats vs Dogs
- **Тип данных**: Изображения
- **Размер датасета**: 25,000 изображений для обучения, 12,500 для тестирования
- **Источник данных**: Kaggle Dogs vs Cats dataset
- **Частота обновления**: Статичный датасет для обучения

### Целевая переменная
- **Название**: label
- **Тип**: Категориальная (binary)
- **Описание**: Класс изображения (кошка или собака)
- **Распределение**: ~50% кошек, ~50% собак

## Схема данных

### Признаки
| Имя признака | Тип | Описание | Обязательный | Примечания |
|-------------|-----|----------|--------------|------------|
| image_path | string | Путь к изображению | да | Уникальный идентификатор |
| pixel_values | array | Пиксельные значения изображения | да | RGB каналы |
| width | int | Ширина изображения | да | В пикселях |
| height | int | Высота изображения | да | В пикселях |
| label | string | Класс изображения | да | "cat" или "dog" |

### Ограничения и валидация
- **Типы данных**: Изображения в формате JPEG/PNG, метки - строки
- **Диапазоны значений**: Ширина/высота > 0, метки только "cat"/"dog"
- **Обязательные поля**: Все поля обязательны
- **Уникальность**: image_path должен быть уникальным

## Качество данных

### Проблемы с данными
- **Пропущенные значения**: Нет (все изображения должны быть полными)
- **Аномалии**: Некоторые изображения могут быть низкого качества
- **Дубликаты**: Возможны дубликаты изображений
- **Несогласованности**: Некоторые изображения могут быть неправильно помечены

### Метрики качества
- **Процент полноты**: 100% (все изображения должны быть доступны)
- **Процент валидности**: >99% (изображения должны быть корректного формата)
- **Процент уникальности**: >99% (минимальное количество дубликатов)
- **Консистентность**: Все изображения должны соответствовать своим меткам

## Требования к предобработке

### Очистка данных
- **Обработка пропущенных значений**: Удаление изображений с некорректными метками
- **Обработка аномалий**: Фильтрация изображений низкого качества
- **Удаление дубликатов**: Сравнение изображений по пиксельным значениям
- **Исправление несогласованностей**: Проверка корректности меток

### Преобразования
- **Нормализация/стандартизация**: Нормализация пикселей к диапазону [0,1]
- **Кодирование категориальных признаков**: One-hot encoding для меток
- **Создание новых признаков**: Извлечение признаков через CNN
- **Снижение размерности**: Изменение размера изображений к фиксированному размеру

## Стратегия разбиения

### Разбиение на выборки
- **Метод разбиения**: Стратифицированное случайное разбиение
- **Пропорции**: 70% обучение, 15% валидация, 15% тест
- **Критерии разбиения**: Случайное разбиение с фиксированным seed
- **Защита от утечки данных**: Разделение до любой предобработки

### Валидация
- **Перекрестная проверка**: 5-fold кросс-валидация
- **Метрики на валидации**: Accuracy, Precision, Recall, F1-score
- **Критерии остановки**: Ранняя остановка при отсутствии улучшения

## Версионирование

- **Стратегия версионирования**: DVC для версионирования датасета
- **Хранение версий**: Хранение хэшей файлов в Git, сами данные в DVC
- **Отслеживание изменений**: Изменения в датасете фиксируются в DVC
